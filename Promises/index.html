Promises

Object that is representing the state

But what state is Promises representing

It is respensenting a eventual completion or failure. A Promise is an object representing the eventual completion (or failure) of an asynchronous operation. Think of it like ordering food at a restaurant — you get a token (promise) that will resolve into food (or fail if they run out).

It has three states

  1. Pending State
  2. Fulfiled
  3. Rejected
